## Step 3: Write at Least 5 Non-Functional Requirements

**What is a Non-Functional Requirement?**
A non-functional requirement describes how the system **should behave** or what qualities it should have. It specifies constraints, performance, security, usability, reliability, etc.

**Characteristics of Good Non-Functional Requirements:**

- Measurable (quantifiable)
- Clear and unambiguous
- Uses "The system shall..." format
- Assign a unique identifier (NFR-XX)

---

### Sample Non-Functional Requirements for Bus Ticketing System

| **ID** | **Non-Functional Requirement**                                                            |
| ------ | ----------------------------------------------------------------------------------------- |
| NFR-01 | The system shall respond to search queries within 3 seconds.                              |
| NFR-02 | The system shall be available 24/7 with a minimum uptime of 99.9%.                        |
| NFR-03 | The system shall support at least 1,000 concurrent users.                                 |
| NFR-04 | The system shall be accessible on both web browsers and mobile devices (Android and iOS). |
| NFR-05 | The system shall be available in both English and Amharic languages.                      |
