# What is a Product Backlog?

---

## Definition

A **Product Backlog** is a **prioritized list** of all the features, enhancements, bug fixes, and tasks that need to be done for a project. It is the single source of truth for what the development team will work on.

---

## Key Characteristics

| **Characteristic** | **Description** |
|--------------------|-----------------|
| Ordered/Prioritized | Items are arranged by priority (highest priority at the top) |
| Living Document | It is continuously updated and refined throughout the project |
| Owned by Product Owner | The Product Owner is responsible for managing the backlog |
| Visible to All | Everyone on the team can see the backlog |
| Items are User Stories | Each item is typically written as a user story |
