
# What is Acceptance Criteria List?

---

## Definition

An **Acceptance Criteria List** is a set of **conditions** that a user story must meet to be considered **complete** and **acceptable** to the user, customer, or product owner.

---

## Key Characteristics

| **Characteristic** | **Description** |
|--------------------|-----------------|
| Specific | Each criterion is clear and unambiguous |
| Testable | Each criterion can be verified (pass/fail) |
| Defines "Done" | It answers the question: "When is this story finished?" |
| Written in Plain Language | Easy for everyone to understand |
| Part of the Story | Usually documented alongside the user story |

---

## Bus Ticketing System Acceptance Criteria List

**User Story:**

> As a **passenger**, I want to **search for bus routes by origin, destination, and date** so that **I can find available buses for my trip**.

**Acceptance Criteria:**

- AC-01: Passenger can enter origin, destination, and travel date
- AC-02: System validates search fields (no empty fields, valid date)
- AC-03: System displays matching routes with times and prices
- AC-04: If no routes found, system shows: "No bus routes found"
- AC-05: Search results are displayed within 3 seconds
