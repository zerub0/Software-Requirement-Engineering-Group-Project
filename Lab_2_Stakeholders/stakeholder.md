
# Stakeholder Roles

| Stakeholder | Role/Description |
|---|---|
| **Passenger** | Searches for bus routes, books tickets, selects seats, makes payments, receives confirmations and reminders. |
| **Bus Operator** | Manages bus schedules, routes, pricing, and seat allocations; tracks revenue. |
| **Bus Company Owner** | Owns the bus company; oversees operations, revenue, and profitability. |
| **Transport Authority** | Government body that regulates bus operations, ensures compliance with safety and pricing regulations. |
| **IT Department** | Installs and maintains the system, manages servers and databases, ensures system security and uptime. |
| **System Developer** | Builds, tests, and deploys the system. |
| **System Tester/QA** | Tests the system for bugs and ensures quality. |
| **Payment Gateway Provider** | Processes online payments (e.g., bank, mobile money, credit card). |
| **Bus Driver** | Operates the bus; receives passenger lists and seating information from the system. |
| **Customer Support Staff** | Handles passenger complaints, refunds, and assistance with bookings. |
| **External Vendors** | Provide hardware, software licenses, or cloud hosting services. |
